## 1. 評価方法

### ラウンドトリップの構成

本来のラウンドトリップテストは以下3フェーズからなる：

```
[Phase 1] PDF → JSON  （LLM API経由・既実施済み）
[Phase 2] JSON → DOCX （to-docxコマンド・本評価で実施）
[Phase 3] DOCX → JSON2（to-jsonコマンド・APIキー未設定のためスキップ）
```

**本評価の範囲**: Phase 2 の品質、すなわち「JSONに含まれるすべてのフィールド値がDOCX文書内に保持されているか」をテキスト照合で評価する。真のラウンドトリップ（JSON1 vs JSON2の比較）ではない点に留意されたい。

### 評価指標

- **総フィールド保持率（単純一致）**: JSONの全文字列値をDOCX全文と部分文字列照合（10文字スライディングウィンドウ）
- **総フィールド保持率（正規化一致）**: 改行・空白を正規化したうえで照合（`\n` による連結問題を吸収）
- **FullText保持率**: JSONの `FullText` フィールド群に特化した保持率

## 5. 発見された問題点

### 問題A: FullText配列のJSON文字列直出し（バグ）
**深刻度: 中**

`OtherPhysicalChemicalProperty[n].AdditionalInfo.FullText` フィールドが配列型（`string[]`）の場合、DOCXにJSON配列リテラル形式で出力される。

**JSONの値**:
```json
{"FullText": ["溶剤: メタノール\n説明: 僅かに溶ける"]}
```

**DOCXの出力**:
```
AdditionalInfo: ["溶剤: メタノール\n説明: 僅かに溶ける"]
```

文字としては保持されているが、`\n` がリテラルのバックスラッシュnとして出力されており、適切な改行表示になっていない。FullText配列は要素を改行またはセミコロンで結合すべきである。

### 問題B: 複数行フィールドの改行削除による連結
**深刻度: 中**

`\n` を含む文字列値（`MaterialsToAvoid`、`HazardousDecompositionProducts.Substance` 等）が、改行なしで一行に連結されて出力される。

**JSONの値**:
```
"強酸化剤\n強力な還元剤"
```

**DOCXの出力**:
```
MaterialsToAvoid: 強酸化剤強力な還元剤
```

セクション区切りなく連続するため、可読性が著しく低下する。

### 問題C: フィールドラベルが英語のまま出力
**深刻度: 高（文書としての品質）**

生成されたDOCXは JIS Z 7253 の 16 セクション見出し（日本語）を持つが、セクション内の各フィールドのラベルがJSONキー名のまま英語で出力される。

**出力例**:
```
CompanyName: テクニカ合同株式会社
Fax: 078-451-0257
FullText: 皮膚刺激あり
```

これでは実務利用に耐えるSDS文書として機能しない。各フィールドには日本語の対応ラベルが必要である。

### 問題D: 表構造（テーブル）が未使用
**深刻度: 中（SDS文書品質）**

11ファイルすべてにおいてDOCX内のテーブル数はゼロ。SDS文書では、組成・濃度情報、物理化学的性質、輸送情報などを表形式で示すことが一般的かつ読みやすい。現在はすべてプレーンテキスト（key: value）として出力されており、文書品質が低い。

### 問題E: JIS JSONセクション欠損（PDF→JSON段階の問題）
**深刻度: 参考情報（to-docxの問題ではない）**

以下のファイルでJSONセクションが欠損しているが、これはPDF→JSON抽出段階の問題であり、`to-docx` コマンドの問題ではない：

## 7. 改善提案

### 優先度: 高

1. **フィールドラベルの日本語化**（問題C対応）  
   JSONキー名から日本語表示名へのマッピングテーブルを実装する。例: `CompanyName` → `会社名`、`FullText` → `（内容）`。

2. **FullText配列の適切な結合**（問題A対応）  
   `string[]` 型の `FullText` フィールドを出力する際、配列要素を改行または句読点で結合する。JSONリテラルのまま出力しない。

### 優先度: 中

3. **複数行フィールドの改行保持**（問題B対応）  
   `\n` を含む文字列値をDOCXに出力する際、DOCX形式のXML段落 (`<w:br/>`) として適切に変換する。

4. **表構造（テーブル）の導入**（問題D対応）  
   組成表（成分名・CAS番号・濃度）、物理化学的性質一覧などをDOCXの `<w:tbl>` 要素として出力することで可読性を大幅向上できる。

### 優先度: 低（将来改善）

5. **APIキーを用いた真のラウンドトリップ評価**  
   本評価はJSON→DOCX変換のみを評価している。DOCX→JSON2変換（LLM再抽出）を含む完全なラウンドトリップ評価により、実際の情報往復品質を測定すべきである。
